# Add session metadata to response-view generated Markdown

## Overview

This ticket tracks adding richer orientation metadata to Markdown generated by `/response-view`, `/rv-last`, and related response-viewer commands. The generated document should include both machine-readable YAML frontmatter and a short human-readable introduction before the assistant response body.

Key requirement: frontmatter paths should be absolute, while links rendered in the Markdown body should be relative links.
