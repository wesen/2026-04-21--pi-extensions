# Selective Session Compaction Extension

## Goal

Create a Pi extension, registered through the existing extension framework, that lets the user mark a **compact start** and **compact end** message in the current session, then request compaction into a new conversation.

## Concise interpretation

Given a session shaped like:

- `system prompt`
- `A`: messages before the selected range
- `compact start marker`
- `B`: selected middle range
- `compact end marker`
- `C`: messages after the selected range

The extension should produce/start a new conversation shaped like:

- `system prompt`
- `A`: original messages before the selected range, preserved verbatim
- `B'`: a summary/compaction of the selected middle range `B`
- `B''`: a generated linkage/bridge message that reconnects the summarized middle to the following context
- `C`: original messages after the selected range, preserved verbatim

## Open validation questions

- Should `B'` be generated by the current model as a normal compaction prompt, or by a dedicated summarization prompt/template owned by the extension?
- Should `B''` be a separate assistant/system-style message, or appended to the summary message?
- Are start/end markers UI-only metadata, visible transcript messages, or both?
- Should compaction create a brand-new conversation immediately, or prepare a proposed transcript for user confirmation first?
